3. Systemd Service Optimization (Critical Learning):
   - ❌ REWRITING SERVICE FILES = DANGEROUS — can crash containers (confirmed by our crash)
   - ✅ DROP-IN CONFIGS in .service.d/ = SAFE — industry standard pattern
   - Pattern: `mkdir -p /etc/systemd/system/code-interpreter.service.d/` then write `10-hardening.conf`
   - This overrides without touching the original service file
   - Debian 13 Trixie (systemd 257) supports: ProtectSystem=strict, NoNewPrivileges=yes, PrivateTmp=yes
   - Use systemd-analyze security to baseline service security scores

4. Cgroups v2 Resource Management:
   - MemoryHigh = soft limit (throttling, service slows but isn't killed)
   - MemoryMax = hard limit (OOM killer activates above this)
   - Both together = safe resource management (back-pressure before OOM)
   - CPUWeight = proportional sharing (1-10000, default 100)
   - IOWeight = proportional I/O priority (replaces IOSchedulingClass for fine-grained control)

5. I/O Scheduling Correction:
   - ❌ IOSchedulingClass=idle — TOO AGGRESSIVE, starves the service of disk I/O
   - ✅ IOWeight=25-100 — Proportional, only throttles under contention
   - IOWeight=50 for background services, IOWeight=500 for critical services (like envd)

6. Docker/Container Tuning (relevant to our environment):
   - userland-proxy=false → iptables directly (kernel handles, more efficient)
   - TCP: tcp_fastopen=3, somaxconn=65535, tcp_tw_reuse=1
   - CPU pinning: cpuset-cpus for cache locality
   - sysctl for high-throughput: net.core.somaxconn=65535, net.core.netdev_max_backlog=65535
